{% set block1="Y" %} {% set block1_title=["Prior mean and standard deviation of the intercept, α0"]%} {% set block1_text=["The intercept refers to the intercept term in the linear regression model for the proximal outcome. The intercept corresponds to the mean of the proximal outcome when the values of the standardized covariates are set to zero."] %} {% set block2="Y" %} {% set block2_title="Normal Prior" %} {% set block2_text %} The algorithm uses a normal distribution to describe prior information about each parameter in the linear regression model—this includes the intercept parameter, α0. Thus to set the prior you must specify a prior mean and a prior standard deviation for each parameter in the linear regression model. {% endset %} {% set block3="Y" %} {% set block3_title="How to set prior values?" %} {% set block3_text %} Using MRT data to inform your choice of prior mean and standard deviation To obtain prior means and standard deviations, we suggest you fit person-specific regression models, one per person in the MRT data (using approximately standardized covariates and proximal outcome). If a person does not have a minimum number of decision points, you might use ridge regression. The estimators as well as their variability across the individual person-specific regression can be used to inform the prior. This approach utilizes the between individual heterogeneity in intercept to proxy the subjective prior uncertainty about the true intercept. And the average of the estimated intercepts across individuals proxies the consensus of the mean for the intercept. Assuming all covariates and the proximal outcome provided to the personalization algorithm will be approximately standardized (so average value of each standardized covariate/proximal outcome is roughly close to 0 and the standard deviation of each standardized covariate/proximal outcome is roughly 1), here are some rules of thumb for setting the prior mean and standard deviation for the intercept:
  • The proximal outcome is approximately standardized: A natural choice for the prior mean of the intercept is 0, and the prior standard deviation is 1.0 (due to the approximate standardization).
  • A second option is to use a weakly informative prior: A prior of Normal (mean = 0, standard deviation = 3.16) is a standard weakly informative prior, which is a widely recommended best practice. Here is what it means:
  • Prior Standard Deviation = 3.16: The value seems specific because its square is approximately 10 (i.e. variance = 10). It expresses uncertainty due to the fact that a different group of individuals are going to experience the pJITAI as compared to individuals in the MRT.
  • [NOTE] When the personalization algorithm is running or actively deployed and receiving real-time data, the proximal outcome (Y) needs to be standardized as well and must utilize the exact same baseline values of the sample mean ($\mu_{Y}$) and variance ($\sigma_(Y)$) used during the original MRT standardization and analysis. Do not recalculate or update $\mu_{Y}$ or $\sigma_(Y)$ using incoming post-deployment data, as shifting these reference points will alter the scaling of your proximal outcome.
