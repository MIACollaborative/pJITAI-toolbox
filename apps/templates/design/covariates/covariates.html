{% set block2="Y" %} {% set block2_title="Covariates in HeartSteps" %} {% set block2_text%} The personalized HeartSteps Activity Suggestions intervention component used two covariates:
  • Previous 30-minute step count (log transformed and standardized in the same way as the proximal outcome)
  • Location (1=[Home, Work], 0=other location)
The previous 30-minute step count is not a tailoring variable, as it is assumed (and was observed in the MRT study) to correlate similarly with proximal outcomes both when a message is sent and when it is not. The location, on the other hand, is a tailoring variable, as it was found to moderate the effect of sending a message versus not sending a message on the proximal outcome. {% endset %} {% set block3="Y" %} {% set block3_title="Covariates and Tailoring Variables" %} {% set block3_text %} Covariates Your algorithm will use covariates describing the individual’s internal state (e.g. mood) and external context (e.g. location, weather) when deciding between intervention options and when updating its model based on proximal outcomes associated with each decision. The information about an individual's internal state and external context is encoded as covariates in the pJITAI toolbox, which are variables you will provide to your algorithm along with every decision request. For example, you might provide the “previous 30 minute step count” as a covariate, based on the assumption that activity just before a decision point will correlate with the proximal outcome of step count after the decision point, regardless of whether the high-intensity option was chosen or not. Tailoring Variables Some covariates might also be tailoring variables. In contrast to non-tailoring covariates, tailoring variables are assumed to moderate the effect of the intervention options on the proximal outcome. For example, the individual’s location, if encoded as a binary covariate where 1=[Home,Work] and -1=[Other location], might be assumed to moderate the effect of the Activity Suggestion (i.e., Activity Suggestions may be more likely to be acted upon when at home or at work than when at, say, the grocery store). Standardization The algorithm will expect that the proximal outcome is approximately standardized. That is, the proximal outcome should be coded so that the average value of the proximal outcome will be roughly 0 and the standard deviation of the proximal outcome will be roughly 1. If the proximal outcome is binary then the proximal outcome might be coded using -1,1 instead of the usual 0,1. A rough rule-of-thumb for standardizing is to guess the usual range of the proximal outcome. Then the average value of the proximal outcome is roughly the center of this range and the standard deviation is roughly the length of the range divided by 4.
  • [NOTE] When the personalization algorithm is running or actively deployed and receiving real-time data, the covariates should be standardized using the same values you used here!
